Froala Configuration for Comments

This demo uses the following Froala Editor configuration optimized for comment sections:

JavaScript Configuration 
new FroalaEditor('#froala-editor', {
      key: 'YOUR_LICENSE_KEY',
      
      // Load plugins useful for comments
      pluginsEnabled: [
        'align', 'charCounter', 'colors', 'emoticons',
        'entities', 'image', 'link', 'lists', 'paragraphFormat',
        'quote', 'url', 'wordCounter', 'mention'
      ],
      
      // Minimal toolbar for cleaner UI
      toolbarButtons: {
        'moreText': {
          'buttons': ['bold', 'italic', 'underline', 'strikeThrough', 'textColor', 'backgroundColor', 'clearFormatting']
        },
        'moreParagraph': {
          'buttons': ['align', 'formatUL', 'formatOL', 'quote']
        },
        'moreRich': {
          'buttons': ['insertLink', 'insertImage', 'emoticons']
        },
        'moreMisc': {
          'buttons': ['undo', 'redo']
        }
      },
      
      // Editor height settings
      heightMin: 150,
      heightMax: 400,
      
      // Placeholder
      placeholderText: 'Write a comment...',
      
      // Image Handling - Simplified for comments
      imageInsertButtons: ['imageBack', '|', 'imageUpload', 'imageByURL'],
      imageEditButtons: ['imageReplace', 'imageAlign', 'imageRemove'],
      imageAllowedTypes: ['jpeg', 'jpg', 'png', 'gif'],
      imageMaxSize: 2 * 1024 * 1024, // 2MB max
      
      // Character count
      charCounterCount: true,
      charCounterMax: 1000,
      
      // Toolbar position
      toolbarBottom: true, // Toolbar at bottom often easier for comments
      
      // Mentions (Example configuration)
      // Requires server-side implementation for user lookup
      /*
      mention: {

          // callback([{value: 'username', id: 1}, ...])
        }
      }
